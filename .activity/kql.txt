

## Coverage Summary And Production Scope

### What This Pack Covers

- Azure and Microsoft security telemetry health: Log Analytics table freshness, Sentinel health, Sentinel audit drift, incident quality, watchlist hygiene, and diagnostic coverage for critical Azure resources.
- Identity and access attack paths: Conditional Access weakening, privileged role/RBAC drift, OAuth consent and app credential persistence, device code/ROPC/legacy authentication, AiTM/token replay patterns, service principal abuse, and Graph/SaaS impact.
- Cloud control plane and compute abuse: logging blind spots followed by impact, Azure VM run command or extension abuse, VMSS/extension drift, Azure management changes, backup weakening, disk/snapshot risk, and resource deletion or defense evasion.
- Network and public edge exposure: NSG, VNet, subnet, route table, public IP, NAT Gateway, Load Balancer, Azure Firewall, Firewall Policy, Azure Front Door, Application Gateway/WAF, APIM, Bastion, Private Endpoint, Private DNS, DDoS, Network Security Perimeter, Network Manager, and flow-log coverage.
- Data, secrets, and storage risk: Key Vault/Managed HSM control weakening followed by secret/key reads, storage public access/shared key/SAS drift, database firewall/audit weakening, data-platform exfiltration, and large object access patterns.
- Application, integration, and recovery surfaces: App Service, Functions, Logic Apps, Container Apps, Automation Accounts, Event Hub, Service Bus, Event Grid, Storage Queue, Backup/Recovery Services, disks, snapshots, Arc-enabled servers, and Azure Virtual Desktop/Bastion paths.
- Containers, DevOps, AI/API, business abuse, and threat intelligence: AKS exec/secret/RBAC activity, ACR image-to-deployment chains, Azure DevOps tampering, workload identity federation abuse, APIM LLM/MCP activity, Azure AI/Search abuse, SMS/OTP/API/cost abuse, and STIX threat-intelligence hits across identity, VM, network, edge, API, storage, and data surfaces.

### Production Boundary

- The pack is production-ready only when the tenant has deployable Sentinel analytics definitions, automation/playbook definitions, workbook or dashboard assets, watchlist CSVs, policy/IaC checks, CI/CD validation, schema-test output, and test incident evidence.
- The KQL in this file is the source logic and acceptance standard. The release package must convert approved controls into YAML/ARM/Bicep/Terraform or the tenant's content-management format.
- The pack does not replace tenant-specific schema validation, licensing checks, diagnostic setting validation, cost review, privacy review, or baseline tuning against real production data.
- The pack does not prove maliciousness by itself. Many rules detect risky sequences or suspicious correlations and still require analyst validation, owner confirmation, and approved-change review.

### Naming Convention Used Below

The `Magma name` field uses a normalized, portable rule-name style: `magma.azure.<layer>.<control_id>.<short_detection_name>`. Keep it lowercase, use dots for hierarchy, and use underscores inside the short detection name.

## Senior Manager Operating Model

### Executive Outcomes

| Outcome | Good looks like | Bad looks like |
|---|---|---|
| Visibility | Every critical identity, workload, data store, network edge, and DevOps path has telemetry or approved exception | Unknown telemetry gaps, silent critical tables, unowned resources |
| Control integrity | Privilege, logging, security, network, and data-protection drift is detected and assigned | Risky changes happen without owner, approval, or rollback evidence |
| Attack detection | Identity abuse, token theft, OAuth persistence, service principal abuse, data access, ransomware prep, and supply-chain attacks create incidents | Alerts are isolated, noisy, and not tied to impact |
| Response readiness | P0 has owner, evidence, containment option, business impact, and closure criteria | P0 becomes an email thread with no accountable action |
| Continuous improvement | False positives, missed detections, table gaps, and exceptions age into backlog work | Rules rot and dashboards look healthy because nobody checks them |

### Severity Model

| Priority | Meaning | Required response |
|---|---|---|
| P0 | Active control failure on critical service, privileged identity risk, defense evasion, production exposure, confirmed data/credential impact, ransomware prep, or multi-surface threat-intelligence hit | Same business day incident, named owner, containment or rollback decision, service-risk note |
| P1 | High-risk posture drift, suspicious activity without confirmed impact, degraded coverage, or unvalidated risky change | Triage within SLA, owner and due date, tuning or exception decision |
| P2 | Hygiene trend, early warning, low-confidence anomaly, or backlog item | Weekly review, backlog owner, aging threshold |

P0 Sentinel custom details must include: `ControlId`, `CoverageLayer`, `Priority`, `Owner`, `ManagerAction`, `AffectedAccount`, `AffectedResource`, `SubscriptionId`, `Evidence`, `FirstSeen`, `LastSeen`, `BusinessService`, `ExceptionId`, `ValidationStatus`.

## Deployment Gates

Do not enable a detection as an analytics rule until its gate passes.

| Gate | Requirement |
|---|---|
| G1 - Table exists | Every required table has recent data or a documented exception |
| G2 - Required columns exist | Columns used by the query are present in the tenant schema |
| G3 - Entity mapping | Account, IP, host, URL, cloud app, and Azure resource fields are mapped |
| G4 - Owner routing | Incident has owner resolution through tags, CMDB, watchlist, or subscription owner |
| G5 - False positive review | Top false-positive causes and approved suppression keys are documented |
| G6 - Test incident | A controlled test creates an incident with expected evidence |
| G7 - Automation safety | Any containment automation has manual approval or confirmed-compromise guardrails |
| G8 - Cost review | Log source, retention, Basic/Auxiliary plan behavior, and ingestion cost are approved |

- Rule name: Required Table Freshness
- Magma name: `magma.azure.schema.schema_001.required_table_freshness`
- Description: Checks whether required Sentinel/Log Analytics tables have recent data before dependent analytics rules are enabled.

### SCHEMA-002 - Watchlist Hygiene

**Rule details**

- Rule name: Watchlist Hygiene
- Magma name: `magma.azure.schema.schema_002.watchlist_hygiene`
- Description: Finds required governance watchlists with empty owners, expired rows, or duplicate keys.
- Simple explanation: This protects routing, suppression, and approval logic. If the watchlists are stale, the detections may assign incidents to the wrong owner or suppress real risk.
- ATT&CK tactics/techniques: Not applicable; maps to governance data quality.
- Rule config: Run daily; query period current watchlist state; severity P1 unless a critical watchlist is unusable, then escalate manually.
- Noise possibility: Medium when teams bulk-load watchlists, migrate CMDB data, or use shared owner aliases.
- Tuning: Add tenant-specific watchlists, normalize owner aliases, allow temporary migration duplicates, and enforce expiry dates through change management.

```kql
let NowTime = now();
union isfuzzy=true
(
    _GetWatchlist("CriticalAssets")
    | extend Watchlist="CriticalAssets", Key=tostring(ResourceId), Owner=tostring(Owner), Expiry=todatetime(Expiry)
),
(
    _GetWatchlist("PrivilegedIdentities")
    | extend Watchlist="PrivilegedIdentities", Key=tostring(AccountUPN), Owner=tostring(Owner), Expiry=todatetime(Expiry)
),
(
    _GetWatchlist("ApprovedSecurityChanges")
    | extend Watchlist="ApprovedSecurityChanges", Key=tostring(ChangeId), Owner=tostring(Approver), Expiry=todatetime(EndTime)
),
(
    _GetWatchlist("ApprovedServicePrincipals")
    | extend Watchlist="ApprovedServicePrincipals", Key=tostring(AppId), Owner=tostring(Owner), Expiry=todatetime(Expiry)
),
(
    _GetWatchlist("SuppressionRegistry")
    | extend Watchlist="SuppressionRegistry", Key=strcat(ControlId, ":", Scope), Owner=tostring(Owner), Expiry=todatetime(EndTime)
)
| summarize Rows=count(), EmptyOwner=countif(isempty(Owner)), Expired=countif(isnotnull(Expiry) and Expiry < NowTime), Duplicates=count() - dcount(Key) by Watchlist
| where EmptyOwner > 0 or Expired > 0 or Duplicates > 0
| project TimeGenerated=NowTime, Priority="P1", ControlId="SCHEMA-002", CoverageLayer="Governance",
          Watchlist, Rows, EmptyOwner, Expired, Duplicates,
          ManagerAction="Fix watchlist quality before relying on owner routing, approvals, or suppressions."
```

### HLT-001 - Critical Table Freshness And Volume Drop

**Rule details**

- Rule name: Critical Table Freshness And Volume Drop
- Magma name: `magma.azure.health.hlt_001.critical_table_freshness_volume_drop`
- Description: Detects silent or sharply reduced telemetry volume for critical security tables.
- Simple explanation: The rule compares recent ingestion with a short baseline. It catches broken connectors, stopped diagnostic settings, DCR issues, agent problems, or product integration failures.
- ATT&CK tactics/techniques: Defense Evasion - Impair Defenses (T1562), Impair Defenses: Disable or Modify Tools (T1562.001).
- Rule config: Run every 30 minutes; query period 15 days; alert P0 for required critical tables that go silent and P1 for optional or volume-drop cases.
- Noise possibility: Medium to high for seasonal workloads, weekends, low-volume tenants, new connectors, Basic logs, and planned maintenance.
- Tuning: Tune `ExpectedTables`, `MaxSilent`, baseline days, and drop threshold per table; suppress approved maintenance windows with expiring suppressions.

```kql

    "AGWAccessLogs", "Public edge", 2h, false,

    "AGWPerformanceLogs", "Public edge", 4h, false,
    "ApiManagementGatewayLogs", "API", 2h, false,
    "AZFWNetworkRule", "Network security", 2h, false,
    "AZFWApplicationRule", "Network security", 2h, false,
    "AZFWNatRule", "Network security", 2h, false,
    "AZFWThreatIntel", "Network security", 4h, false,
    "AZFWIdpsSignature", "Network security", 4h, false,
    "NatGatewayFlowlogsV1", "Network egress", 4h, false,
    "ALBHealthEvent", "Load balancer", 24h, false,
    "MicrosoftAzureBastionAuditLogs", "Remote administration", 24h, false,
    "NSPAccessLogs", "Network Security Perimeter", 4h, false,
    "AKSAudit", "Containers", 4h, false,
```

- Noise possibility: Medium for stopped VMs, ephemeral scale-set instances, golden images, maintenance windows, decommissioning, and hostname/resource-ID mismatch.
- Tuning: Keep `CriticalAssets` and VM tags current, exclude intentionally stopped/deallocated VMs, normalize resource IDs, and require owner-approved telemetry exceptions.

```kql
let CriticalVms =
    _GetWatchlist("CriticalAssets")
    | where AssetType in~ ("VM", "VirtualMachine", "VMSS", "ScaleSet", "Server")
    | project ResourceId=tolower(ResourceId), AssetName=tolower(AssetName), BusinessService, Owner, Environment, Criticality;
let Mde =
    DeviceInfo
    | summarize arg_max(Timestamp, *) by DeviceId
    | extend ResourceId=tolower(tostring(column_ifexists("AzureResourceId", ""))),
             AzureVmId=tostring(column_ifexists("AzureVmId", "")),
             SensorHealth=tostring(column_ifexists("SensorHealthState", "")),
             Onboarding=tostring(column_ifexists("OnboardingStatus", "")),
             Exposure=tostring(column_ifexists("ExposureLevel", ""))
    | summarize LastMde=max(Timestamp), DeviceNames=make_set(DeviceName, 10), SensorStates=make_set(SensorHealth, 10), OnboardingStates=make_set(Onboarding, 10), ExposureLevels=make_set(Exposure, 10) by ResourceId;
let Guest =
    union isfuzzy=true
    (Heartbeat | where TimeGenerated > ago(7d) | extend ResourceId=tolower(tostring(column_ifexists("_ResourceId", ""))) | project TimeGenerated, ResourceId, Signal="Heartbeat"),
    (InsightsMetrics | where TimeGenerated > ago(7d) | extend ResourceId=tolower(tostring(column_ifexists("_ResourceId", ""))) | project TimeGenerated, ResourceId, Signal="InsightsMetrics"),
    (Perf | where TimeGenerated > ago(7d) | extend ResourceId=tolower(tostring(column_ifexists("_ResourceId", ""))) | project TimeGenerated, ResourceId, Signal="Perf")
    | where isnotempty(ResourceId)
    | summarize LastGuest=max(TimeGenerated), GuestSignals=make_set(Signal, 10) by ResourceId;
CriticalVms
| join kind=leftouter Mde on ResourceId
| join kind=leftouter Guest on ResourceId
| extend MissingMde=isempty(LastMde) or LastMde < ago(24h),
         MissingGuest=isempty(LastGuest) or LastGuest < ago(24h),
         BadSensor=SensorStates has_any ("Inactive", "Impaired communications", "No sensor data") or OnboardingStates has_any ("Can be onboarded", "Unsupported", "Insufficient info")
| where MissingMde or MissingGuest or BadSensor or ExposureLevels has "High"
| extend Priority=iff(Criticality in~ ("critical", "high") or Environment =~ "prod", "P0", "P1")
| project TimeGenerated=now(), Priority, ControlId="HLT-016", CoverageLayer="Azure VM",
          AzureResourceCustomEntity=ResourceId, AssetName, BusinessService, Owner, Environment, Criticality,
          LastMde, LastGuest, DeviceNames, SensorStates, OnboardingStates, ExposureLevels, GuestSignals,
           ManagerAction="Azure VM/VMSS telemetry is incomplete. Restore MDE, AMA/Heartbeat/VM Insights, or approved exception before relying on VM detections."
```

### HLT-017 - Component Coverage Evidence Health

**Rule details**

- Rule name: Component Coverage Evidence Health
- Magma name: `magma.azure.health.hlt_017.component_coverage_evidence_health`
- Description: Finds critical Azure components that are listed as covered but lack owner, expected telemetry, approved baseline, or freshness evidence.

- Tuning: Populate component-specific watchlists first, then set required evidence by component class and criticality. Do not suppress missing owner plus missing telemetry together.

```kql
        | project ResourceId=tolower(ResourceId), Component=tostring(AssetType), BusinessService, Owner, Environment, Criticality,

                  ExpectedDiagnostics="", ExpectedBaseline="SensitiveAppServices"),
    (_GetWatchlist("ApprovedModelDeployments")
        | project ResourceId=tolower(ResourceId), Component="AIModelDeployment", BusinessService="", Owner, Environment="", Criticality=tostring(DataSensitivity),
                  ExpectedDiagnostics="APIM/AppInsights/model logs", ExpectedBaseline="ApprovedModelDeployments");
let ObservedUsage =
    Usage
    | where TimeGenerated > ago(CoverageLookback)
    | summarize LastIngest=max(TimeGenerated), DataTypes=make_set(DataType, 500), VolumeMb=sum(Quantity)
    | extend JoinKey=1;

| extend JoinKey=1

| where MissingOwner or MissingExpectedDiagnostics or MissingIngestion or ExpiredOrWeakBaseline
```

### PRO-020 - Crown-Jewel Path Opened

Detect a newly reachable path into a `CrownJewelWorkloads` resource by joining RBAC, network exposure, private endpoint, DevOps identity, and approved-change data. Escalate when a path is new, unapproved, and lacks owner/expiry.

### PRO-021 - Deployment Identity Blast Radius

Detect CI/CD or IaC identities that can modify identity, network, secrets, compute, and deployment resources across the same production scope. Split privileges when one identity can both create access and deploy impact.

### PRO-022 - Secretless Expectation Broken

Detect apps marked `SecretlessExpected=true` that receive connection strings, client secrets, publishing credentials, account keys, or non-Key-Vault app settings. Treat the drift as credential exposure until rotated.

### PRO-023 - Public-To-Private Boundary Regression

Detect sensitive stores or apps expected to be private-only that gain public network access, broad firewall rules, public DNS, public IPs, or origin exposure. Require immediate owner approval or rollback.

### PRO-024 - Admin Device Trust Mismatch

Detect privileged Azure actions from devices absent from `PrivilegedCloudWorkstations`, failing MDE health, or missing compliant-device evidence. Use this as a precondition for session revocation and privileged access review.

### PRO-025 - Over-Permissioned Workload Identity

Detect service principals and managed identities whose Azure RBAC, Graph app roles, or Key Vault/data permissions exceed `ApprovedServicePrincipals` and `ApprovedDeploymentIdentities` scope.

### PRO-026 - Egress Intent Mismatch

Detect critical workloads sending traffic to destinations not listed in `AllowedEgressDestinations`, especially after network, DNS, NAT, APIM, or firewall policy drift.

### PRO-027 - Recovery Fragility

Detect production workloads whose delete locks, backup immutability, soft delete, purge protection, or recovery-point health would not meet business RPO/RTO during destructive attack.

### PRO-028 - API Abuse Economics

Detect API, SMS, and AI endpoints where expected request volume, token use, countries, spend caps, and throttling controls are insufficient for abuse resistance.

### PRO-029 - Identity-To-Data Chain Exposed

Detect newly privileged users, apps, or managed identities that can reach sensitive data without an approved app path, private network path, or owner-approved data-access pattern.

### PRO-030 - Control-Owner Dead Zone

Detect critical resources with missing owner, expired exception, missing diagnostics, missing incident route, and recent risky activity. Block production risk acceptance until ownership is fixed.

### DET-001 - CA Or Auth Policy Weakened Followed By Privileged Sign-In

**Rule details**

- Rule name: CA Or Auth Policy Weakened Followed By Privileged Sign-In
- Magma name: `magma.azure.detection.det_001.ca_auth_policy_weakened_privileged_signin`
- Description: Correlates Conditional Access or authentication-policy weakening with privileged sign-ins shortly afterward.
- Simple explanation: A weakened policy is urgent when a privileged identity immediately benefits from it.
- ATT&CK tactics/techniques: Defense Evasion - Impair Defenses (T1562), Valid Accounts (T1078), Account Manipulation (T1098).
- Rule config: Run every 15 minutes; query period 24 hours; severity P0.
- Noise possibility: Medium during IAM release windows, break-glass exercises, and CA rollout projects.
- Tuning: Join approved IAM changes, privileged identity watchlist, and trusted admin network ranges.

### DET-028 - Network Exposure Or Inspection Bypass Followed By Allowed Flow

**Rule details**

- Rule name: Network Exposure Or Inspection Bypass Followed By Allowed Flow
- Magma name: `magma.azure.detection.det_028.network_exposure_bypass_followed_by_allowed_flow`
- Description: Correlates risky Azure network changes with allowed VNet/NSG/firewall flows to sensitive ports or critical destinations.
- Simple explanation: A route, NSG, public IP, peering, firewall, AFD, or AGW change is more urgent when traffic starts flowing through the new path soon after.
- ATT&CK tactics/techniques: Initial Access - External Remote Services (T1133), Exploit Public-Facing Application (T1190), Defense Evasion - Impair Defenses (T1562), Exfiltration Over Web Service (T1567).
- Rule config: Run every 15-30 minutes; query period 24 hours; severity P0 for production critical destinations or admin/database ports.
- Noise possibility: Medium to high during network releases, failover tests, blue/green migrations, and firewall policy deployments.
- Tuning: Use `ApprovedNetworkFlows`, `ApprovedNetworkExposures`, `AzureNetworkAssets`, expected scanners, and production criticality to suppress known paths only.

```kql
let Lookback = 24h;

    AzureActivity
    | where TimeGenerated > ago(Lookback)
    | where ActivityStatusValue =~ "Success"
    | extend Provider=toupper(tostring(column_ifexists("ResourceProviderValue", ""))),

             ResourceIdLower=tolower(ResourceId),
             Raw=tolower(tostring(Properties))
    | where Provider == "MICROSOFT.NETWORK" or ResourceIdLower has "/providers/microsoft.network/"
    | where Op has_any ("write", "delete")
    | where Raw has_any ("0.0.0.0/0", "internet", "allow", "disabled", "bypass", "publicipaddress", "routetable", "virtualnetworkpeering", "firewall", "waf", "privateendpointnetworkpolicies")
    | project ChangeTime=TimeGenerated, Actor=tolower(tostring(Caller)), ActorIP=CallerIpAddress, OperationNameValue, ChangedResource=ResourceIdLower, SubscriptionId, ResourceGroup, Raw;
let AllowedFlow =
    union isfuzzy=true
    (NTANetAnalytics
        | where TimeGenerated > ago(Lookback)
        | where tostring(column_ifexists("SubType", "")) =~ "FlowLog" or isempty(tostring(column_ifexists("SubType", "")))
        | extend SourceIp=tostring(column_ifexists("SrcIp", "")),
                 DestinationIp=tostring(column_ifexists("DestIp", "")),
                 DestinationPort=toint(column_ifexists("DestPort", int(null))),
                 Protocol=tostring(column_ifexists("L4Protocol", "")),
                 FlowStatus=tostring(column_ifexists("FlowStatus", "")),
                 FlowDirection=tostring(column_ifexists("FlowDirection", "")),
                 FlowCount=tolong(column_ifexists("FlowCount", 1)),
                 SubscriptionId=tostring(column_ifexists("_SubscriptionId", ""))
        | where FlowStatus has_any ("Allow", "Allowed", "A") or FlowDirection =~ "Inbound"
        | project FlowTime=TimeGenerated, Surface="VNetFlow", SourceIp, DestinationIp, DestinationPort, Protocol, FlowStatus, FlowDirection, FlowCount, SubscriptionId),
    (AZFWNetworkRule
        | where TimeGenerated > ago(Lookback)
        | where Action =~ "Allow"
        | project FlowTime=TimeGenerated, Surface="AzureFirewallNetwork", SourceIp, DestinationIp, DestinationPort, Protocol, FlowStatus=Action, FlowDirection="", FlowCount=1, SubscriptionId=_SubscriptionId),
    (AZFWNatRule
        | where TimeGenerated > ago(Lookback)

    | where array_index_of(SensitivePorts, DestinationPort) >= 0 or FlowDirection =~ "Inbound";
NetworkChange
| join kind=inner AllowedFlow on SubscriptionId
| where FlowTime between (ChangeTime .. ChangeTime + 6h)
| summarize Flows=sum(FlowCount), Sources=make_set(SourceIp, 50), Destinations=make_set(DestinationIp, 50), Ports=make_set(DestinationPort, 50), Surfaces=make_set(Surface, 10),
            FirstFlow=min(FlowTime), LastFlow=max(FlowTime)
    by ChangeTime, Actor, ActorIP, OperationNameValue, ChangedResource, SubscriptionId, ResourceGroup, Raw
| where Flows >= 5 or array_length(set_intersect(Ports, SensitivePorts)) > 0
| project TimeGenerated=LastFlow, Priority="P0", ControlId="DET-028", CoverageLayer="Network security",
          AccountCustomEntity=Actor, IPCustomEntity=ActorIP, AzureResourceCustomEntity=ChangedResource,
          OperationNameValue, SubscriptionId, ResourceGroup, Flows, Sources, Destinations, Ports, Surfaces, FirstFlow, LastFlow, Raw,
          ManagerAction="Network exposure or inspection bypass was followed by allowed flow. Validate change approval, block or restrict path, and inspect destination service."
```

### DET-029 - Azure VM Remote Admin Or Management Execution Followed By Host Impact

**Rule details**

- Rule name: Azure VM Remote Admin Or Management Execution Followed By Host Impact
- Magma name: `magma.azure.detection.det_029.azure_vm_remote_admin_management_execution_host_impact`
- Description: Correlates Azure VM run command, extension, Bastion/remote admin, backup weakening, or disk/snapshot changes with suspicious host behavior.
- Simple explanation: This turns isolated VM management activity into an impact-driven incident when the guest shows suspicious process, network, logon, or recovery-inhibition behavior.
- ATT&CK tactics/techniques: Execution - Cloud Administration Command (T1651), Command and Scripting Interpreter (T1059), Valid Accounts (T1078), Inhibit System Recovery (T1490), Credential Dumping (T1003).
- Rule config: Run every 15 minutes; query period 24 hours; severity P0 when host impact is present, P1 for suspicious management activity without host confirmation.
- Noise possibility: Medium for patching, deployment agents, troubleshooting, backup lifecycle tasks, and vulnerability scanners.
- Tuning: Use approved deployment identities, `ApprovedRemoteAdminPaths`, VM criticality, known scripts, and maintenance windows.

```kql
let Lookback = 24h;

        | extend VmName=tolower(extract(@"/virtualMachines/([^/]+)", 1, ResourceId))
        | project ActionTime=TimeGenerated, Surface="AzureActivity", Actor=tolower(tostring(Caller)), ActorIP=CallerIpAddress, Operation=OperationNameValue, ResourceId=tolower(ResourceId), VmName, Evidence=tostring(Properties)),

        | where TimeGenerated > ago(Lookback)

        | where Attempts >= 20 or Failed >= 10
        | project ImpactTime=LastLogon, VmName, ImpactType="RemoteLogonAnomaly", ImpactEvidence=strcat("Attempts=", Attempts, "; Failed=", Failed, "; Accounts=", tostring(Accounts), "; RemoteIPs=", tostring(RemoteIPs)))

VmManagement
| join kind=leftouter HostImpact on VmName
| extend HasHostImpact=isnotempty(FirstImpact) and FirstImpact between (ActionTime .. ActionTime + 6h)
| where HasHostImpact or Operation has_any ("RUNCOMMAND", "EXTENSIONS/WRITE", "BACKUPPROTECTEDITEMS/DELETE", "BACKUPPOLICIES/DELETE")
| extend Priority=iff(HasHostImpact, "P0", "P1")
| project TimeGenerated=coalesce(LastImpact, ActionTime), Priority, ControlId="DET-029", CoverageLayer="Azure VM",
          AccountCustomEntity=Actor, IPCustomEntity=ActorIP, HostCustomEntity=VmName, AzureResourceCustomEntity=ResourceId,
          Surface, Operation, Evidence, HasHostImpact, ImpactTypes, ImpactEvidence, FirstImpact, LastImpact,
          ManagerAction="Azure VM management or remote admin activity has host-impact context. Validate admin intent, isolate if suspicious, preserve evidence, and restore backup/security controls."
```

### DET-030 - Network Threat Intelligence Match Across Azure Edge And Flow

**Rule details**

- Rule name: Network Threat Intelligence Match Across Azure Edge And Flow
- Magma name: `magma.azure.detection.det_030.network_ti_match_azure_edge_flow`
- Description: Matches active STIX indicators from `ThreatIntelIndicators` against Azure Firewall, AFD, AGW, APIM, VNet flow, NAT Gateway, and VM egress observations, with optional context from `ThreatIntelObjects`.
- Simple explanation: This is the Azure-network TI rule. It prioritizes indicators seen on multiple network surfaces, allowed flows, successful API/edge responses, or VM egress.
- ATT&CK tactics/techniques: Command and Control (TA0011), Initial Access (TA0001), Exfiltration (TA0010), Impact (TA0040), depending on indicator and surface.
- Rule config: Run every 15-30 minutes; query period 24 hours with 30-day TI lookback; severity P0 for high confidence, allowed/successful impact, VM egress, or multi-surface match.
- Noise possibility: High for shared hosting, CDN IPs, scanners, sinkholes, stale OSINT, and low-confidence feeds.
- Tuning: Enforce confidence/expiry/TLP, suppress known benign CDN/shared services, require allow/success impact for low-confidence indicators, and prefer exact URL/domain/IP normalization.

```kql
let Lookback = 24h;

let TiObjects =
    ThreatIntelObjects
    | where TimeGenerated > ago(TiLookback)
    | where IsDeleted == false

    | where isnotempty(ObjectStixId) and StixType !in~ ("relationship", "indicator")
    | project ObjectStixId, ObjectContext=strcat(StixType, ":", ObjectName);
let TiRelationships =
    ThreatIntelObjects
    | where TimeGenerated > ago(TiLookback)
    | where IsDeleted == false and StixType =~ "relationship"
    | extend DataDynamic=todynamic(Data), SourceRef=tostring(DataDynamic.source_ref), TargetRef=tostring(DataDynamic.target_ref), RelationshipType=tostring(DataDynamic.relationship_type)
    | extend IndicatorStixId=iff(SourceRef startswith "indicator--", SourceRef, iff(TargetRef startswith "indicator--", TargetRef, "")),
             RelatedStixId=iff(SourceRef startswith "indicator--", TargetRef, SourceRef)
    | where isnotempty(IndicatorStixId) and isnotempty(RelatedStixId)
    | join kind=inner TiObjects on $left.RelatedStixId == $right.ObjectStixId
    | summarize RelatedObjects=make_set(ObjectContext, 20), Relationships=make_set(RelationshipType, 20) by IndicatorStixId;
let ActiveTI =
    ThreatIntelIndicators
    | where TimeGenerated > ago(TiLookback)
    | where IsActive == true and IsDeleted == false and Revoked != true
    | where isnull(ValidUntil) or ValidUntil > now()
    | where Confidence >= 60
    | extend IndicatorValue=tolower(tostring(ObservableValue)),
             ObservableKey=tolower(tostring(ObservableKey)),
             IndicatorStixId=tostring(todynamic(Data).id),
             TLP=tostring(parse_json(tostring(column_ifexists("AdditionalFields", ""))).TLPLevel)
    | extend IndicatorType=case(
        ObservableKey has "ipv4-addr" or ObservableKey has "ipv6-addr", "IP",
        ObservableKey has "domain-name" or ObservableKey has "hostname", "Domain",
        ObservableKey has "url", "Url",
        "Other")
    | where IndicatorType != "Other" and isnotempty(IndicatorValue)
    | summarize arg_max(TimeGenerated, *) by Id, IndicatorType, IndicatorValue
    | join kind=leftouter TiRelationships on IndicatorStixId
    | project TiId=Id, IndicatorType, IndicatorValue, Confidence, Tags, TLP, Pattern, ValidUntil, IndicatorStixId, RelatedObjects, Relationships;
let Observed =
    union isfuzzy=true
    (AZFWNetworkRule | where TimeGenerated > ago(Lookback) | project HitTime=TimeGenerated, Surface="AzureFirewallNetwork", ObservedType="IP", ObservedValue=tolower(DestinationIp), SourceIp, DestinationIp, DestinationPort, Url="", ResourceId=_ResourceId, Impact=Action),

    (AZFWApplicationRule | where TimeGenerated > ago(Lookback) and isnotempty(Fqdn) | project HitTime=TimeGenerated, Surface="AzureFirewallApp-Domain", ObservedType="Domain", ObservedValue=tolower(Fqdn), SourceIp, DestinationIp, DestinationPort, Url=Fqdn, ResourceId=_ResourceId, Impact=Action),
    (AZFWThreatIntel | where TimeGenerated > ago(Lookback) | project HitTime=TimeGenerated, Surface="AzureFirewallTI", ObservedType="IP", ObservedValue=tolower(DestinationIp), SourceIp, DestinationIp, DestinationPort, Url=Fqdn, ResourceId=_ResourceId, Impact=Action),
    (ApiManagementGatewayLogs | where TimeGenerated > ago(Lookback) | project HitTime=TimeGenerated, Surface="APIM-IP", ObservedType="IP", ObservedValue=tolower(CallerIpAddress), SourceIp=CallerIpAddress, DestinationIp="", DestinationPort=int(null), Url, ResourceId=_ResourceId, Impact=iff(ResponseCode between (200 .. 299), "APIMSuccess", tostring(ResponseCode))),
    (AGWAccessLogs | where TimeGenerated > ago(Lookback) | project HitTime=TimeGenerated, Surface="AGW-ClientIP", ObservedType="IP", ObservedValue=tolower(ClientIp), SourceIp=ClientIp, DestinationIp="", DestinationPort=int(null), Url=RequestUri, ResourceId=_ResourceId, Impact=tostring(HttpStatus)),
```

### DET-031 - Admin Device Trust Mismatch Followed By Azure Impact

Use the advanced pattern with `PrivilegedCloudWorkstations`, sign-in device context, MDE health, and AzureActivity impact. Escalate when privileged actions come from unmanaged, unhealthy, or unknown devices.

### DET-032 - Workload Identity Federation Drift Followed By Cloud Use

Use `ApprovedFederatedCredentials`, AuditLogs, service principal sign-ins, and AzureActivity. Escalate when issuer, subject, audience, repository, branch, or environment changes before successful cloud use.

### DET-033 - App Service Or Function Secretless Regression

Use `SensitiveAppServices`, app settings/config changes, publishing credential events, Key Vault reads, and data access logs. Escalate when a secretless workload gains secrets and then accesses sensitive resources.

### DET-034 - Automation Runbook Or Webhook Persistence

Use `ApprovedAutomationRunbooks`, automation job logs, webhook/credential changes, and AzureActivity. Escalate when a modified runbook or webhook performs privileged changes.

### DET-035 - Private DNS Or Private Endpoint Hijack

Use private endpoint approvals, private DNS record/link changes, sensitive data-store watchlists, and data-plane logs. Escalate when path changes are followed by access from new sources.

### DET-036 - DCR Or AMA Blind Spot Plus Host Impact

Use DCR association activity, Heartbeat/AMA freshness, MDE process/network/logon events, and VM criticality. Escalate when visibility is removed before host impact.

### DET-037 - Data Factory, Synapse, Or Databricks External Sink Exfil

Use pipeline/job/notebook audit, linked service changes, sensitive data stores, and storage/database data-plane logs. Escalate when external sinks or copy jobs appear near high-volume reads.

### DET-038 - Messaging Drain, Replay, Or Poison Flood

Use Event Hub, Service Bus, Event Grid, Storage Queue, SAS policy/key changes, and message metrics/logs. Escalate when identity/key drift aligns with receive/delete/dead-letter/publish anomalies.

### DET-039 - Azure AI Or Search Index Data Exposure

Use `ApprovedModelDeployments`, APIM/App Insights/Search/Cognitive logs, key operations, content-filter state, and token/query/download baselines. Escalate on public endpoint drift plus sensitive responses or surge.

### DET-040 - AVD Or Bastion Risky Admin Session Followed By Workload Impact

Use AVD/Bastion audit, risky sign-ins, MDE host impact, VM commands, and data access. Escalate when risky remote admin is followed by cloud or host impact.

### DET-041 - Backup Vault Weakening Plus Snapshot Or Disk Export

Use Recovery Services operations, immutable vault state, disk/snapshot actions, grants, exports, and critical asset mapping. Escalate when recovery controls weaken before disk or snapshot access.

### DET-042 - Arc Extension Command Execution Followed By Hybrid Host Impact

Use Arc extension operations, Guest Configuration drift, MDE/AMA signals, and AzureActivity. Escalate when Azure control-plane actions cause suspicious activity on Arc-connected hosts.

### DET-043 - API Key Or Subscription Key Spray With Eventual Success

Use APIM, App Service, AGW, App Insights, and approved API client baselines. Escalate when repeated failed key/client attempts become successful against sensitive endpoints.

### DET-044 - Origin Bypass Behind Front Door Or WAF

Use AFD/AGW/WAF logs, app-origin logs, NSG/firewall flows, and origin exposure inventory. Escalate when direct-to-origin traffic appears after WAF/front-door drift.

### DET-045 - Custom Domain, DNS, Or Certificate Takeover Drift

Use DNS zone changes, custom-domain bindings, managed cert/TLS changes, App Service/AFD/CDN/APIM logs, and approved changes. Escalate unapproved domain or certificate path changes.

### DET-046 - Cost Spike With Matching API, AI, SMS, Compute, Or Egress Abuse

Use cost exports, APIM/AI/SMS/NAT/compute metrics, campaign calendars, and owner routing. Escalate when spend spikes align with abuse telemetry and no approved launch or load test exists.

### DET-047 - Cross-Tenant Guest Or B2B Path To Production

Use cross-tenant access settings, guest invitations, access packages, RBAC, sign-ins, and data access. Escalate when new external trust reaches production resources.

### DET-048 - Break-Glass Account Used Outside Break-Glass Procedure

Use `PrivilegedIdentities`, sign-ins, AzureActivity, active incidents, and approved emergency changes. Escalate any break-glass sign-in without an active incident and named owner.

### DET-049 - Resource Lock Removal Followed By Destructive Operation

Use management lock activity, critical assets, deletion operations, backup state, and approved changes. Escalate when lock removal precedes deletion, backup weakening, or data destruction.

### DET-050 - Control-Owner Dead Zone Exploited

Use `CriticalAssets`, owner tags, diagnostics, exceptions, incident routing, and attack/drift signals. Escalate when an unowned or expired-exception resource receives risky activity.

### DET-031 to DET-050 Implementation Pattern

Use this pattern for every advanced extension so the rules stay reliable and manager-actionable:

```kql
// Pattern: Drift or suspicious precondition + impact + owner context.
let Lookback = 24h;
let CrownJewels =
    _GetWatchlist("CrownJewelWorkloads")
    | project CrownResourceId=tolower(ResourceId), WorkloadName, BusinessService, Owner, Tier, PrimaryIdentity, DataStores, IngressPaths, EgressPolicy;

    _GetWatchlist("ApprovedSecurityChanges")
    | project ApprovedActor=tolower(Actor), ApprovedResource=tolower(ResourceId), ApprovedOperation=tolower(Operation),
              StartTime=todatetime(StartTime), EndTime=todatetime(EndTime), ChangeId;
let Precondition =
    AzureActivity
    | where TimeGenerated > ago(Lookback)
    | where ActivityStatusValue =~ "Success"
    | extend Actor=tolower(tostring(Caller)), ResourceIdLower=tolower(ResourceId), Operation=tolower(OperationNameValue), Evidence=tolower(tostring(Properties))
    | where Operation has_any ("write", "delete", "action")
    | where Evidence has_any ("publicnetworkaccess", "diagnosticsettings/delete", "roleassignments", "federatedidentitycredentials", "webhook", "secret", "privateendpoint", "dns", "backup", "lock")
    | project PreTime=TimeGenerated, Actor, ActorIP=CallerIpAddress, ResourceId=ResourceIdLower, Operation, Evidence, SubscriptionId;
let Impact =
    union isfuzzy=true
    (AzureActivity | where TimeGenerated > ago(Lookback) and ActivityStatusValue =~ "Success" | where OperationNameValue has_any ("DELETE", "ROLEASSIGNMENTS", "KEYVAULT", "STORAGE", "NETWORK", "COMPUTE", "BACKUP") | project ImpactTime=TimeGenerated, ImpactActor=tolower(tostring(Caller)), ImpactResource=tolower(ResourceId), ImpactSurface="AzureActivity", ImpactEvidence=OperationNameValue),
    (StorageBlobLogs | where TimeGenerated > ago(Lookback) | summarize Requests=count(), Bytes=sum(tolong(column_ifexists("ResponseBodySize", 0))), Last=max(TimeGenerated) by ImpactResource=tolower(_ResourceId), CallerIpAddress | where Requests > 1000 or Bytes > 1073741824 | project ImpactTime=Last, ImpactActor="", ImpactResource, ImpactSurface="Storage", ImpactEvidence=strcat("Requests=", Requests, ";Bytes=", Bytes)),

Precondition
| join kind=leftouter ApprovedChanges on $left.Actor == $right.ApprovedActor
| extend Approved=isnotnull(StartTime) and PreTime between (StartTime .. EndTime)
| where not(Approved)
| extend JoinKey=pack_array(Actor, ResourceId)
| mv-expand JoinKey to typeof(string)
| join kind=inner (Impact | extend JoinKey=pack_array(ImpactActor, ImpactResource) | mv-expand JoinKey to typeof(string)) on JoinKey
| where ImpactTime between (PreTime .. PreTime + 8h)
| join kind=leftouter CrownJewels on $left.ResourceId == $right.CrownResourceId
| extend Priority=iff(Tier in~ ("0", "1", "critical") or ImpactSurface in ("Storage", "MDE"), "P0", "P1")
| project TimeGenerated=ImpactTime, Priority, ControlId="DET-031-050", CoverageLayer="Cross-surface attack path",
          AccountCustomEntity=Actor, IPCustomEntity=ActorIP, AzureResourceCustomEntity=coalesce(CrownResourceId, ResourceId, ImpactResource),
          WorkloadName, BusinessService, Owner, Tier, Operation, Evidence, ImpactSurface, ImpactEvidence, ChangeId,
          ManagerAction="Cross-surface precondition was followed by impact. Validate approval, contain affected path, restore control, and document business impact."
```

## Component-Complete Attack Extensions

DET-051 to DET-070 are the impact-correlated rules for the two expanded coverage matrices. They should not be deployed as generic noise rules. Each rule must join a risky precondition to a successful flow, backend impact, data access, host signal, or spend/business impact.

| Detection | Required precondition | Required impact evidence | Manager action |
|---|---|---|---|
| DET-051 - Internet admin/database path opened and hit | PRO-031/034 change or ARG shows public admin/database path to critical asset | VNet/NSG flow, Firewall DNAT, AGW/LB admin NAT, VM logon, or DB login success on sensitive port | Close path or approve with expiry, isolate destination if suspicious, inspect credentials and host/data access |
| DET-052 - NSG/ASG lateral movement, scan, or brute-force allowed after rule drift | NSG/ASG allow, ASG membership drift, or NIC/subnet NSG mismatch | Many sources/ports, SMB/RDP/SSH/WinRM/SQL/Redis/Kubernetes flow, failed-then-success logon, or MDE lateral tool evidence | Revert rule, block source, reset affected credentials, inspect peer hosts |
| DET-053 - Peering, UDR, VPN, or ExpressRoute bypass followed by east-west/data flow | PRO-032/033/041 reachability or route drift | New east-west flow, on-prem-to-critical flow, data-plane read/copy, or firewall inspection gap | Restore route/peering intent, validate BGP/route propagation, inspect accessed workload |
| DET-054 - Public IP, DDoS, admin targeting, or TI source against exposed workload | PRO-034 public IP/DDoS drift or new public attachment | DDoS metric/log, inbound scan, admin targeting, WAF/FW hit, TI source, or backend failures | Attach DDoS plan/edge protection, restrict public path, activate incident if service impact exists |
| DET-055 - Firewall TI/IDPS/DNS/C2 or blocked-to-allowed sequence | PRO-035 policy weakening, allow/DNAT added, TI/IDPS/DNS disabled | `AZFWThreatIntel`, `AZFWIdpsSignature`, suspicious DNS, allowed C2 flow, or same tuple blocked then allowed | Re-enable prevention/TI/IDPS, block IOC, inspect source workload and downstream impact |
| DET-056 - AFD/AppGW WAF exploit wave with origin/backend impact | PRO-036 WAF/origin drift or new WAF hit surge | WAF OWASP/bot signatures, direct-origin request, backend 5xx/latency/auth failures, app error spike | Restore WAF prevention/origin privacy, block source, gather app exploit evidence |

## Production Rule Standard

Every analytics rule must include:

| Field | Requirement |
|---|---|
| Rule name | `Priority - ControlId - Human readable detection` |
| Description | Attack/control intent, required data, and primary response |
| Tactics/techniques | ATT&CK mapping where applicable |
| Frequency/query period | P0 5-15 minutes, P1 30-60 minutes, posture daily if ARG |
| Entity mapping | Account, IP, host, URL, Azure resource, cloud app where available |
| Custom details | Required P0 fields plus evidence and owner |
| Suppression | Scoped to account/resource/control, never global |
| Incident grouping | Group by account/resource/source IP within short window |
| Automation | Enrich by default; contain only on confirmed compromise or explicit approval |
| Validation | Test incident ID, sample evidence, expected owner, pass/fail status |

## Production Deployment Artifact Checklist

| Analytics-as-code | One YAML/ARM/Bicep/Terraform definition per enabled rule, with query, schedule, severity, entity mappings, custom details, grouping, suppression, and incident settings |

5. Validate Azure network inventory: NSG/ASG, VNet/subnet, peering, route table/UDR, public IP/DDoS, NAT, LB, Firewall/Firewall Policy, AFD, AGW/WAF, APIM, Bastion, Private Endpoint/DNS, NSP, AVNM, VPN, and ExpressRoute.
6. Validate VM inventory: VM/VMSS criticality, MDE onboarding, AMA/Heartbeat, VM Insights, backup, update posture, disk/snapshot handling, and remote admin paths.
7. Validate expanded component inventory: management groups, Policy, Defender, Monitor/DCR/AMA, Entra governance, app platforms, Automation, Key Vault/HSM, databases, storage/data platforms, messaging, AKS/ACR, DevOps, AVD/Bastion, AI/Search/ML, ACS, Backup/Recovery, disks/snapshots, and Arc.
8. Validate TI migration: all production TI rules use `ThreatIntelIndicators` and `ThreatIntelObjects`; legacy `ThreatIntelligenceIndicator` is migration-only.
9. Enable rules in monitor-only mode by layer: SecOps, identity, Azure VM, network, data, DevOps, containers, AI/API, business abuse, TI, and component-complete coverage.
10. Tune thresholds against at least 30 days of tenant data.
11. Create test incidents for every P0 detection with expected entity mapping and custom details.
12. Validate owner routing from tags, watchlists, subscription ownership, network asset ownership, or CMDB.
13. Validate PRO-031 to PRO-045 using approved change windows and known-bad drift samples.
14. Validate DET-051 to DET-070 using paired precondition-plus-impact simulations, not isolated posture events.
15. Review top false positives and create scoped, expiring suppressions only.
16. Enable enrichment automation first; enable containment automation only after tabletop approval.
17. Review coverage, cost, false positives, missed cases, and expired exceptions monthly.

## Known Limitations

- KQL execution cannot be guaranteed across tenants until table and column gates pass.
- Azure Resource Graph is required for complete inventory and effective configuration coverage.
- Some diagnostic logs can be Basic or lake-only and may not support all analytics use cases.
- NSG flow logs are a migration/debt area for tenants that still use them. New coverage should prefer virtual network flow logs and traffic analytics where supported.
- AFD and some legacy network resources may emit through `AzureDiagnostics` with category-specific columns; schema gates must validate category columns before enablement.
- Azure-side VPN/ExpressRoute telemetry is covered, but on-premises router/firewall logs are out of scope for this Azure-only pack.
- `ThreatIntelligenceIndicator` is legacy for this pack. Use `ThreatIntelIndicators` and `ThreatIntelObjects` for production TI detections.
- Many cloud detections are impact-correlated, not proof of maliciousness by themselves.
- Exact TI matching must be supplemented with normalization, enrichment, allowlists, and analyst review.
- Some AI/API detections rely on APIM gateway logging and may need privacy review before request/response logging is enabled.
- Component-complete detections require ARG/effective-route/DNS/policy exports; log-only detections cannot prove full reachability or route intent.
- VPN/ExpressRoute rules cover Azure-side changes and Azure-observed traffic only unless on-premises device telemetry is explicitly onboarded.
- Some automated response actions can cause outages. Use enrichment-first automation and require approval for containment unless compromise is confirmed.
